# Авито AI Ассистент для объявлений

## Технологии

- **Node.js:** v20+
- **Frontend:** React 18, TypeScript, Vite
- **UI:** Mantine UI
- **State Management:** Redux Toolkit + RTK Query
- **Routing:** React Router DOM
- **AI:** Ollama (локальная LLM)
- **HTTP Client:** RTK Query
- **Линтер:** ESLint
- **Форматтер:** Prettier

## Запуск проекта

### 1. Клонирование репозитория

```bash
git clone https://github.com/nndog1337/avito.git
cd avito
```

### 2. Настройка и запуск бэкенда
**Откройте новый терминал** и выполните:
```bash
cd avito/server
npm install
npm start
```

Сервер запустится на http://localhost:8080

> **Примечание:** В бэкенд были внесены следующие изменения:
> - Добавлено поле `id` в ответ эндпоинта `GET /items` (список объявлений)
> - Настроен CORS для поддержки запросов с фронтенда
> - Добавлена обработка `OPTIONS` запросов
> - Добавлены обязательные поля "Тип" для всех категорий

### 3. Настройка AI (Ollama)
Для работы AI-функций необходимо установить Ollama:

Windows:

Скачайте установщик с ollama.com

Установите Ollama

Запустите Ollama (он появится в трее)

После установки скачайте модель:

```bash
ollama pull llama3.2
```
Запустите сервер Ollama:

```bash
ollama serve
```
Сервер AI будет доступен на http://localhost:11434

### 4. Запуск фронтенда

**Откройте новый терминал** и выполните:
```bash
cd avito
npm install
npm run dev
```
Приложение запустится на http://localhost:5173

Принятые решения
1. Модификация бэкенда
В исходном бэкенде эндпоинт GET /items не возвращал id объявлений, что делало невозможной навигацию на детальную страницу. Было принято решение:

Добавить поле id в ответ GET /items

Настроить CORS для корректной работы с фронтендом

2. Выбор стейт-менеджмента
Использован Redux Toolkit + RTK Query вместо рекомендованного Zustand + React Query, так как:

RTK Query предоставляет встроенное кеширование и инвалидацию данных

Единая экосистема Redux упрощает поддержку кода

Автоматическая типизация из коробки

3. Выбор UI-библиотеки
Выбран Mantine UI благодаря:

Встроенной поддержке тёмной темы

Адаптивным компонентам

Простоте кастомизации

Качественной документации

4. Обработка отсутствия id в списке
Данные детальной страницы передаются через state React Router, что позволяет:

Избежать дополнительных запросов при клике из списка

Обеспечить мгновенную загрузку страницы

Сохранить работоспособность при прямом переходе по URL

5. Автосохранение черновика
Реализовано с debounce (500 мс) для оптимизации количества записей в localStorage. Черновик очищается после успешного сохранения или при отмене редактирования.

6. Обязательные поля
В форме редактирования обязательными являются:

Название

Цена (должна быть больше 0)

Тип (КПП для авто, тип недвижимости, тип устройства)

В случае отсутствия значения отображается сообщение об ошибке под полем.
